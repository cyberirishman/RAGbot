# === Defensive Transitive Pins (v1.6 hardening) ===
# These three packages are pulled in transitively by other libraries (LangChain
# pulls pydantic; ChromaDB and others pull numpy). They are pinned explicitly
# here because version drift in any of them has historically caused breaking
# changes that look mysterious from a user's perspective:
#   - pydantic 2.13+ emitted JSON schemas that older gradio_client couldn't
#     parse, crashing the Gradio UI on first request (fixed in v1.5.1).
#   - NumPy 2.x is a major version with breaking API changes that affect
#     ChromaDB, LangChain, scikit-image, and transformers.
# Pinning these keeps a fresh install reproducible even if LangChain or another
# upstream library bumps its dependency requirements in the future.
pydantic==2.12.3               # Schema validation core for LangChain
pydantic_core==2.41.4          # Compiled core for pydantic; tightly coupled
numpy==1.26.4                  # NumPy 1.x — must NOT be 2.x (breaking changes)
